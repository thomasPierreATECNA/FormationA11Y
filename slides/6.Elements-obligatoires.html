Les éléments obligatoires

Thématique 8 du RGAA.

Le RGAA définit 10 critères pour s'assurer de la présence des éléments indispensables à la compréhension d'une page web, notamment par les lecteurs d'écran.

Voir les 10 critères

--- layout: default transition: slide-up ---

Les éléments obligatoires

La langue principale de la page

Critères 8.3 et 8.4

Comme vous le savez, certaines personnes ont recours à des lecteurs d'écran pour naviguer sur le web. Ces outils convertissent le texte en parole.

Il est très important de déclarer la langue principale de la page avec l'attribut lang afin que les lecteurs d'écran puissent correctement interpréter et prononcer le contenu de façon claire.

Les éléments obligatoires

Changement de langue dans la page

Critère 8.7 et 8.8

Dans le cas où une partie du contenu est rédigée dans une langue différente de la langue principale de la page, il est essentiel de l'indiquer.

On utilisera alors egalment l'attribut lang sur l'élément concerné.

```html ...

Je suis un texte en français.

I am a text in English.

```

Les éléments obligatoires

Le sens de lecture

Critère 8.10

Tout comme la langue, le sens de lecture doit être correctement indiqué.

Le sens de lecture par défaut est de gauche à droite (ltr). Si la page contient une langue qui se lit de droite à gauche, il faut l'indiquer via l'attribut dir .

```html ...

Je suis un texte en français.

أنا نص باللغة العربية

```

Les éléments obligatoires

Le titre la page

Critère 8.5 et 8.6

Chaque page web doit posséder un titre pertinent et idéalement unique qui reflète son contenu.

C'est ce titre qui s'affiche dans l'onglet et l'historique du navigateur et qui est affiché dans les résultats des moteurs de recherche.

Un titre de page bien rédigé aide les utilisateurs à comprendre le contenu de la page avant même de l'ouvrir, et facilite la navigation, en particulier pour les personnes utilisant des lecteurs d'écran.

Les éléments obligatoires

Le titre la page

Bien que ce ne soit pas obligatoire, il est fortement recommandé d'indiquer également le nom du site dans le titre de page.

Cela permet de mieux s'y retrouver sur plusieurs onglets ouverts ou dans l'historique du navigateur.

Quelques exemples :

```html {*}{lines:false} Accueil du site | Decathlon ``` ```html {*}{lines:false} Chaussure de running KALENJI 500 | Decathlon ``` ```html {*}{lines:false} Mon compte client | Decathlon ``` ```html {*}{lines:false} Mon compte client - Adresse de livraison | Decathlon ```

Les éléments obligatoires

Le titre la page

Sur une page de recherche, le terme recherché doit obligatoirement apparaître

```html {*}{lines:false} Résultats de recherche pour "Gants de boxe" | Decathlon ```

Si la page contient une pagination, le numéro de page et le nombre de page total doivent également apparaître

```html {*}{lines:false} Résultats de recherche pour "Gants de boxe" - page 2 sur 18 | Decathlon ```

Les éléments obligatoires

Le respect de la semantique

Critère 8.9

HTML 5 propose plus de 100 balises. Il est fortement conseillé d'utiliser les balises sémantiques pour structurer le contenu et ainsi faciliter la compréhension de la page par les technologies d'assistance.

D'un autre côté, l'utilisation de balise sémantiques à des fins de présentation est interdite. Par exemple, on n'utilisera pas une balise de titre pour afficher un texte en gros si ce n'est pas un titre de section.
